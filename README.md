# متابعة التواصل — Outreach Tracker

موقع بسيط (HTML/CSS/JS) + Supabase لإدارة التواصل مع الصالونات/العيادات/الفنادق،
مع تتبّع لحظي لمنع إن اتنين من الفريق يبعتوا لنفس الرقم.

## المميزات
- تسجيل دخول للفريق (Supabase Auth).
- جدول بكل الأرقام: الاسم / الرقم / المدينة / الموقع / التصنيف / الحالة.
- زر **إرسال** يفتح واتساب بالرسالة جاهزة، وبيحجز الرقم باسمك قبل الإرسال.
- تحديث الحالة: اتبعت / ردّ / مفيش رد.
- تحديث **لحظي** (Realtime) — الكل يشوف التغييرات فورًا.
- **رفع CSV** مع اختيار التصنيف (أو إضافة تصنيف جديد) + **تجاهل المكرر تلقائيًا**.
- فلاتر + بحث + إحصائيات.
- يشتغل في **وضع تجريبي** ببيانات وهمية قبل ما توصّله بـ Supabase.

## التشغيل (3 خطوات)

### 1) Supabase
1. اعمل مشروع جديد على https://supabase.com
2. افتح **SQL Editor > New query**، والصق محتوى `schema.sql` كله ثم Run.
3. من **Authentication > Users** أضف حسابات لأعضاء الفريق (بريد + كلمة مرور).

### 2) config.js
افتح `config.js` وحط الـ **Project URL** والـ **anon public key**
(من Supabase: Project Settings > API). وعدّل كود الدولة والرسالة لو حبيت.

### 3) GitHub Pages
1. ارفع الملفات دي على ريبو.
2. Settings > Pages > Deploy from branch > main / root.
3. افتح الرابط وسجّل دخول.

## صيغة ملف CSV
أول صف عناوين الأعمدة (مدعوم عربي وإنجليزي):

```
name,phone,city,location
صالون النخبة,01001234567,القاهرة,مدينة نصر
```

- التصنيف (صالون/عيادة...) بتختاره من الموقع وقت الرفع، مش محتاج عمود في الملف.
- `phone` لوحده إجباري، والباقي اختياري.
- الأرقام بتتظبط تلقائيًا للصيغة الدولية (01xxxx -> 201xxxx).

## منع التكرار
عمود `phone` في قاعدة البيانات `unique`، والرفع بيستخدم `upsert ... ignoreDuplicates`،
يعني أي رقم موجود قبل كده بيتم تجاهله وما بيتأثرش وضعه الحالي.
